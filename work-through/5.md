# Work-through 5: Device Registration & Admin Approval

## Mục tiêu

Triển khai flow đăng ký và phê duyệt thiết bị để đảm bảo chỉ thiết bị được admin/manager chấp thuận mới có thể chấm công.

---

## Vấn đề cần giải quyết

Khi app check WiFi, nếu thiết bị chưa được đăng ký hoặc chưa được phê duyệt, cần chặn chấm công và yêu cầu đăng ký thiết bị.

**Flow:**
```
App mở → Check WiFi hiện tại
  └─ BSSID không khớp branch_wifis → Lỗi WIFI_NOT_ALLOWED
  └─ BSSID khớp → Check device
       └─ Chưa tồn tại → POST /v1/devices/register → status=pending
       └─ status=pending → Lỗi DEVICE_PENDING
       └─ status=rejected → Lỗi DEVICE_REJECTED
       └─ status=approved → Cho phép chấm công
```

**Phân tích DB cũ:**
- `devices` chỉ có `is_trusted bool` — không có trạng thái duyệt/từ chối, không có metadata thiết bị
- Thiếu `device_name`, `platform`, `model` để admin biết duyệt thiết bị nào
- Thiếu `approved_by`, `approved_at` để audit

### 1. Migration `202603290001` — `cmd/migration/table.go`

Thêm migration mới vào cuối danh sách:

```go
ID: "202603290001"

// Xóa: is_trusted
// Thêm index: idx_devices_status
// Thêm FK: user_id → users ON DELETE CASCADE, approved_by → users
```

**Lý do xóa `is_trusted`:** Trường boolean không đủ để mô tả trạng thái `pending`. Thay bằng `status varchar(20)` với 3 giá trị: `pending`, `approved`, `rejected`.

### 2. Model — `internal/models/device.go`

**Trước:**
```go
type Device struct {
    ID        uint
    UserID    *uint
    DeviceID  string
    IsTrusted *bool
    CreatedAt time.Time
}
```

**Sau:**
```go
type DeviceStatus string

const (
    DeviceStatusPending  DeviceStatus = "pending"
    DeviceStatusApproved DeviceStatus = "approved"
    DeviceStatusRejected DeviceStatus = "rejected"
)

type Device struct {
    ID         uint
    UserID     *uint
    DeviceID   string       `gorm:"varchar(100)"`
    DeviceName string       `gorm:"varchar(200)"`
    Platform   string       `gorm:"varchar(20)"`   // android / ios
    Model      string       `gorm:"varchar(100)"`
    Status     DeviceStatus `gorm:"varchar(20);default:'pending'"`
    ApprovedBy *uint
    ApprovedAt *time.Time
    CreatedAt  time.Time
}
```

**Request types mới:**
```go
type RegisterDeviceRequest struct {
    DeviceID   string `json:"device_id" validate:"required"`
    DeviceName string `json:"device_name"`
    Platform   string `json:"platform"`
    Model      string `json:"model"`
}

type DeviceStatusQuery struct {
    DeviceID string `query:"device_id" validate:"required"`
}

type DeviceListQuery struct {
    Status string `query:"status"`
}
```

### 4. Service — `internal/services/device_service.go` (file mới)

```go
type DeviceService struct {
    repo *repositories.DeviceRepository
}
```

| Method | Mô tả |
|--------|-------|
| `Register(ctx, userID, req)` | Đăng ký thiết bị. Nếu đã tồn tại → trả về record cũ. Nếu mới → tạo với `status=pending` |
| `GetStatus(ctx, userID, deviceID)` | Lấy trạng thái thiết bị của user hiện tại |
| `GetByUserID(ctx, userID)` | Danh sách thiết bị của user |
| `ListByStatus(ctx, status)` | Admin: lọc thiết bị theo status |
| `Approve(ctx, adminID, id)` | Admin: duyệt thiết bị → `status=approved` |
| `Reject(ctx, adminID, id)` | Admin: từ chối thiết bị → `status=rejected` |

### 5. Handler — `internal/handlers/device_handler.go` (file mới)

| Endpoint | Method | Handler | Mô tả |
|----------|--------|---------|-------|
| `POST /v1/devices/register` | Register | Đăng ký thiết bị (tất cả user đã login) |
| `GET /v1/devices/status?device_id=` | GetStatus | Kiểm tra trạng thái thiết bị |
| `GET /v1/devices` | ListMyDevices | Danh sách thiết bị của tôi |
| `GET /v1/admin/devices?status=` | AdminList | Admin: danh sách thiết bị, filter theo status |
| `PUT /v1/admin/devices/:id/approve` | Approve | Admin: phê duyệt |
| `PUT /v1/admin/devices/:id/reject` | Reject | Admin: từ chối |

**Role check trong admin endpoints:**
```go
role, _ := c.Get("role").(string)
if role != "admin" && role != "manager" {
    return response.Error(c, appErrors.ErrForbidden)
}
```
