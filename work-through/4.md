# Walkthrough: Flutter Mobile App Implementation

## Summary

Implemented the full Flutter mobile app for [daily-hello-app](file:///Users/hoangit/source/daily-hello/daily-hello-app) based on [app_struct.md](file:///Users/hoangit/source/daily-hello/daily-hello-app/app_struct.md). The app is a smart attendance client supporting Check-in/Check-out via GPS + WiFi, attendance history, and a personal dashboard.

Architecture: **Feature-based + MVVM (Provider)**

## Layer Breakdown

### core/

**`api_client.dart`** — Khởi tạo Dio với `BaseOptions` (baseUrl, timeout). Interceptor tự động đọc token từ `SecureStorage` và gắn vào header `Authorization: Bearer <token>`.

**`secure_storage.dart`** — Wrapper của `FlutterSecureStorage`. Cung cấp `saveToken`, `getToken`, `deleteToken`, `hasToken`.

**`app_constants.dart`** — Định nghĩa `baseUrl = http://localhost:8080/api`, key names cho storage.

**`date_format_utils.dart`** — Static helpers: `formatDate`, `formatTime`, `formatDateTime`, `formatDuration` (tính khoảng cách check-in → check-out).

---

### models/

**`user.dart`** — Fields: `id`, `username`, `fullName`, `email`, `role`, `branchId`. Factory `fromJson` map backend response.

**`attendance.dart`** — Fields: `id`, `userId`, `checkIn`, `checkOut?`, `lat`, `lng`, `wifiSsid`, `wifiBssid`, `status`. Getter `isCheckedOut` tiện dùng trong UI.

**`branch.dart`** — Fields: `id`, `name`, `branchCode`, `parentBranchCode`, `address`, `lat`, `lng`, `wifiSsids`.

---

### services/

Mỗi service chỉ gọi API, không chứa business logic.

**`auth_service.dart`**
- `login(username, password)` → POST `/auth/login` → trả `{token, user}`
- `getProfile()` → GET `/auth/me` → trả `User`
- `logout()` → POST `/auth/logout`

**`attendance_service.dart`**
- `checkIn({lat, lng, wifiSsid, wifiBssid})` → POST `/attendance/check-in`
- `checkOut({lat, lng, wifiSsid, wifiBssid})` → POST `/attendance/check-out`
- `getHistory({from, to, page, limit})` → GET `/attendance` với pagination
- `getTodayAttendance()` → GET `/attendance/today`

**`branch_service.dart`**
- `getBranches()` → GET `/branches`
- `getBranch(id)` → GET `/branches/:id`

---

### features/

#### auth/

**`auth_controller.dart`** — `ChangeNotifier`. State: `currentUser`, `isLoading`, `errorMessage`.
- `login()`: gọi service → lưu token → parse user
- `loadProfile()`: load lại user từ API
- `logout()`: xóa token, reset state
- `isLoggedIn()`: check token tồn tại trong storage

**`login_page.dart`** — Form đăng nhập với validation. Username + password (toggle visibility). Hiển thị lỗi từ controller. Sau login thành công → navigate `/home`.

---

#### attendance/

**`attendance_controller.dart`** — `ChangeNotifier`. State: `todayAttendance`, `history`, `isLoading`, `isLoadingHistory`, `currentPage`, `hasMore`.
- `checkIn()` / `checkOut()`: lấy GPS qua `Geolocator.getCurrentPosition()`, lấy WiFi qua `NetworkInfo`, gửi lên backend.
- `loadHistory({refresh})`: load theo trang, append vào list; `refresh=true` reset pagination.
- `_getPosition()`: kiểm tra quyền GPS, request nếu chưa có.
- `_getWifiInfo()`: bắt lỗi nếu không lấy được WiFi (silent fail).

**`checkin_page.dart`** — Hiển thị trạng thái hôm nay (check-in time, check-out time, tổng giờ). Nút check-in / check-out tùy theo `todayAttendance`. Badge trạng thái: Chưa check-in / Đang làm việc / Đã check-out.

**`history_page.dart`** — ListView với infinite scroll: khi scroll xuống cuối load trang tiếp. Pull-to-refresh. Mỗi item hiển thị ngày, giờ check-in/out, tổng giờ, badge trạng thái.

---

#### dashboard/

**`dashboard_controller.dart`** — `ChangeNotifier`. Gọi song song `getTodayAttendance()` và `getHistory(limit:5)` bằng `Future.wait`. Getter `presentDays` đếm số ngày đầy đủ trong recent history.

**`dashboard_page.dart`** — 3 section:
1. **Greeting card** — Avatar, tên, role, ngày hiện tại (nền primary color)
2. **Today card** — Check-in, check-out, tổng giờ hôm nay
3. **Recent history list** — 5 bản ghi gần nhất, link "Xem tất cả" → `/history`

---

#### profile/

**`profile_controller.dart`** — `ChangeNotifier`. `loadProfile()` lấy user từ API. `logout(context)`: gọi service → xóa token → navigate về `/login` (remove all).

**`profile_page.dart`** — Hiển thị avatar, tên, role badge, info rows (username, email, branch). Nút đăng xuất có confirm dialog.

---

### widgets/

**`app_button.dart`** — `ElevatedButton` full-width (height 48). Props: `label`, `onPressed`, `isLoading`, `icon`, `color`. Khi `isLoading=true` hiển thị `CircularProgressIndicator` nhỏ.

**`app_card.dart`** — `Card` với `InkWell` và border-radius 12. Props: `child`, `padding`, `onTap`, `color`.

---

### app.dart

**`DailyHelloApp`** — `MaterialApp` với Material3, seed color `#1976D2`. Routes: `/login`, `/home`, `/history`.

**`_AuthGate`** — `initState` kiểm tra token → redirect `/login` hoặc `/home`. Hiển thị loading spinner trong khi check.

**`MainShell`** — Bottom navigation với 3 tab: Dashboard / Chấm công / Hồ sơ. Dùng `NavigationBar` (Material3).

---

### main.dart

`AppProviders` — `StatelessWidget` khởi tạo tất cả dependencies thủ công (manual DI, không dùng service locator):
1. `SecureStorage`
2. `ApiClient(secureStorage)`
3. Services (dùng `apiClient.dio`)
4. Controllers (dùng services)

Wrap toàn bộ bằng `MultiProvider` → `DailyHelloApp`.

## App Flow

```
App start
  └─ _AuthGate
       ├─ hasToken? ──Yes──> /home (MainShell)
       └─ No ──────────────> /login

/login
  └─ AuthController.login()
       ├─ success ──> save token ──> /home
       └─ fail   ──> show error

/home (MainShell - BottomNav)
  ├─ Tab 0: DashboardPage
  │    └─ DashboardController.loadDashboard() [Future.wait]
  │         ├─ getTodayAttendance()
  │         └─ getHistory(limit:5)
  ├─ Tab 1: CheckInPage
  │    └─ AttendanceController
  │         ├─ checkIn() ──> GPS + WiFi ──> POST /attendance/check-in
  │         └─ checkOut() ─> GPS + WiFi ──> POST /attendance/check-out
  └─ Tab 2: ProfilePage
       └─ ProfileController.logout() ──> clear token ──> /login
```

## Key Decisions

- **Manual DI** trong `main.dart` thay vì `get_it` — đơn giản, đủ dùng cho scope project.
- **Provider.value** cho services (singleton), **ChangeNotifierProvider** cho controllers (có lifecycle).
- **Silent fail WiFi** — nếu không lấy được SSID/BSSID vẫn cho phép check-in (GPS là chính).
- **Pagination** trong history: append theo trang, `hasMore` flag dừng khi backend trả empty list.
- **`_AuthGate`** tách riêng khỏi `DailyHelloApp` để routing logic không làm phức tạp MaterialApp.
- `flutter analyze` — **No issues found** sau khi hoàn thành.
